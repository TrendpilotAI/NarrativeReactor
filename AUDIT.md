## 3. Security Issues

### ЁЯФ┤ CRITICAL

**JWT sessions lack expiry (`exp` claim)**
- File: `src/lib/jwt.ts`
- Issue: JWT tokens signed without `expiresIn` option
- Fix: Add `expiresIn: '8h'` to `jwt.sign()` options
- Risk: Stolen session tokens are valid indefinitely

**No CSP/security headers**
- File: `src/index.ts`
- Issue: No `helmet` middleware; missing Content-Security-Policy, X-Frame-Options, HSTS
- Fix: `app.use(helmet())` before all routes
- Risk: XSS, clickjacking vulnerabilities in dashboard

### ЁЯЯб MEDIUM

**Some SQLite queries may use string interpolation**
- Audit all files in `src/services/` for template literals in SQL strings
- Ensure all user-supplied values go through parameterized queries

**API key in logs**
- Risk: If structured logging is added carelessly, API keys could appear in logs
- Fix: Add log redaction for `x-api-key`, `authorization` headers in pino

### ЁЯЯв LOW

**DASHBOARD_PASSWORD stored in env as plaintext**
- Acceptable for current scale, but should migrate to bcrypt hash comparison
- File: `src/middleware/dashboardAuth.ts`

## 6. Performance Bottlenecks

### ЁЯФ┤ HIGH

**No response caching for AI generation**
- Every identical prompt тЖТ full AI API call ($$$)
- File: `src/flows/content-generation.ts`
- Fix: LRU cache (hash of prompt+brand тЖТ cached response, 5min TTL)
- Savings: Could reduce AI costs by 30-50% in heavy usage

**Synchronous video generation**
- Video jobs block HTTP thread for minutes
- File: `src/flows/orchestration.ts`, `src/services/videoStitcher.ts`
- Fix: Job queue (SQLite-backed or Bull) + polling endpoint

### ЁЯЯб MEDIUM

**Missing SQLite indexes**
- `content_library` тАФ no index on `brand_id`, `created_at`
- `schedules` тАФ no index on `scheduled_at`, `status`
- `campaigns` тАФ no index on `brand_id`, `phase`

**No pagination on list endpoints**
- `/api/content`, `/api/campaigns`, `/api/schedules` return all rows
- Will degrade with >1000 records

**Full lodash import** тАФ adds ~70KB to bundle unnecessarily
