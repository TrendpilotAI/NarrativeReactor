## 2. DRY Violations

### Publisher logic split
**Files**: `src/services/publisher.ts` + `src/services/blotatoPublisher.ts`

`publisher.ts` contains platform formatters (Twitter threading, LinkedIn formatting) and delegates to `postToSocialFlow`. `blotatoPublisher.ts` calls `formatForPlatform` from publisher but then routes through `blotatoPublish`. Two publishing paths exist simultaneously.

**Fix**: Create `src/services/publishing/` with:
- `formatters.ts` — all platform formatters
- `blotato-adapter.ts` — Blotato-specific publishing
- `social-adapter.ts` — Direct social API publishing
- `index.ts` — Unified `publish(content, platforms, options)` entry point

### Error handling patterns
Each service has its own try/catch with inline `console.error`. No shared error types.

**Files affected**: All 30 services in `src/services/`

**Fix**: Create `src/lib/errors.ts` with `ServiceError`, `PublishError`, `AIGenerationError` etc.

## 3. Security Issues

### CRITICAL: CORS wildcard
**File**: `src/index.ts`, line ~18
```typescript
app.use(cors({ origin: '*' }));
```
In production this allows any origin to call the API. 

**Fix**: 
```typescript
const allowedOrigins = process.env.ALLOWED_ORIGINS?.split(',') || ['http://localhost:3000'];
app.use(cors({ origin: allowedOrigins }));
```

### HIGH: Timing-unsafe webhook secret comparison
**File**: `src/routes/webhooks.ts`, lines 10-19
```typescript
if (provided !== secret) { // String equality — timing attack vulnerable
```

**Fix**:
```typescript
import { timingSafeEqual } from 'crypto';
const a = Buffer.from(provided || '');
const b = Buffer.from(secret);
if (a.length !== b.length || !timingSafeEqual(a, b)) { ... }
```

### MEDIUM: No body size limit
**File**: `src/index.ts`
No `express.json({ limit: '1mb' })` — DoS vector via large payloads.

**Fix**: `app.use(express.json({ limit: '1mb' }));`

### MEDIUM: Webhook endpoint not rate-limited
**File**: `src/index.ts`, line ~48
`/webhooks` routes have no rate limiting, only `/api` routes do.

**Fix**: Apply rate limiter to webhooks with stricter limits (e.g., 30/min per IP).

### LOW: Wildcard dependency versions
**File**: `package.json`
```json
"genkit": "*",
"@genkit-ai/dotprompt": "*",
"genkitx-anthropic": "*"
```
Wildcard `*` accepts any version including breaking/vulnerable releases.

**Fix**: Pin to specific semver ranges after `npm audit`.

### LOW: Missing security headers
No `helmet` middleware. Exposes server info, missing CSP, X-Frame-Options etc.

**Fix**: `npm install helmet` + `app.use(helmet())` in index.ts.

## 4. Performance Bottlenecks

### No caching for brand context
**Files**: `src/lib/context.ts`, used by all flows

`loadStoryBibleContext()` and `loadBrandGuidelines()` are called on every content generation request. These read from disk/Firestore and don't cache results.

**Fix**: Add Redis-backed cache with 5-minute TTL:
```typescript
const cached = await redis.get(`brand:${brandId}`);
if (cached) return JSON.parse(cached);
```

### Synchronous content generation (no job queue)
**Files**: `src/routes/pipeline.ts`, `src/flows/content-generation.ts`

Content generation runs inline in HTTP request handlers. Long-running AI calls (10-30s) hold the HTTP connection open.

**Fix**: BullMQ queue — return `{ jobId }` immediately, poll `/api/jobs/:id` for status.

### In-memory cost tracking
**File**: `src/services/costTracker.ts`

Cost data resets on every restart. No persistence.

**Fix**: Persist to Redis (fast) + Postgres (permanent) on every tracked event.
